# 更新日志

## 2026-04-01 - 自动匹配图片分辨率功能修复

### 问题描述
在虚拟摄像头启动状态下切换图片时，"自动匹配图片分辨率"功能无法正常工作：
- 切换不同分辨率的图片后，摄像头分辨率不会自动更新
- 图片会被强制缩放到原分辨率，导致变形
- OBS虚拟摄像头快速重启时会报错

### 修复内容

#### 1. `virtual_camera.py` - `stop()` 方法
- 添加 `time.sleep(0.5)` 延迟，给OBS虚拟摄像头时间释放资源
- 解决快速重启时资源未完全释放的问题

#### 2. `virtual_camera.py` - `start()` 方法
- 添加 `retry_count` 参数（默认3次重试）
- 每次重试间隔1秒
- 提供更详细的错误日志
- 解决OBS虚拟摄像头偶尔启动失败的问题

#### 3. `gui.py` - `_update_camera_frame()` 方法（完全重写）
- 直接在方法中处理摄像头的停止、更新分辨率、加载图片、重启流程
- 不再依赖 `set_resolution()` 方法
- 正确更新按钮状态（启动/停止按钮）
- 支持摄像头未运行时仅更新分辨率设置

#### 4. `gui.py` - `on_auto_resolution_changed()` 方法
- 使用与 `_update_camera_frame()` 相同的可靠重启流程
- 停止 → 更新分辨率 → 加载图片 → 重启
- 正确处理按钮状态

#### 5. `gui.py` - `apply_resolution()` 方法
- 使用相同的可靠重启流程
- 确保重启后重新加载当前图片
- 正确处理启动失败的情况

### 关键改进

1. **可靠的重启流程**
   - 直接控制 stop → 更新设置 → load_image → start 的顺序
   - 避免依赖 `set_resolution()` 方法的不确定性

2. **资源释放延迟**
   - `stop()` 后等待0.5秒，确保OBS虚拟摄像头完全释放
   - 解决"virtual camera output could not be started"错误

3. **自动重试机制**
   - `start()` 失败时自动重试最多3次
   - 每次重试间隔1秒
   - 提高启动成功率

4. **正确的状态管理**
   - 重启后正确更新按钮状态
   - 提供清晰的状态消息反馈
   - 处理启动失败时的UI状态

5. **避免不必要的重启**
   - 只有分辨率真正不同时才重启摄像头
   - 相同分辨率时直接更新帧数据

### 测试验证

创建了 `test_auto_resolution_fix.py` 测试脚本，验证以下场景：
1. 切换相同分辨率的图片（不重启）
2. 切换不同分辨率的图片（成功重启）
3. 切换不同分辨率的图片（启动失败处理）
4. 自动匹配分辨率禁用时的行为
5. 摄像头未运行时的行为
6. `on_auto_resolution_changed()` 方法的各种场景

所有测试均通过。

### 使用说明

启用"自动匹配图片分辨率"后：
1. 选择图片或从列表中双击图片
2. 系统会自动检测图片分辨率
3. 如果分辨率与当前摄像头不同，会自动重启摄像头
4. 摄像头会使用新图片的分辨率
5. 无需手动停止/启动摄像头

### 注意事项

- 切换不同分辨率的图片时，摄像头会短暂重启（约1-2秒）
- 重启期间视频会议软件可能会显示摄像头断开
- 这是正常行为，重启完成后会自动恢复
